Overview

simple_decimal is a clean wrapper over Gobo's MA_DECIMAL library that provides arbitrary-precision decimal arithmetic for Eiffel. Perfect for financial calculations where binary floating-point errors are unacceptable.

Binary floating-point (REAL, DOUBLE) cannot represent many decimal fractions exactly. The classic example: 0.1 + 0.2 = 0.30000000000000004. With simple_decimal, 0.1 + 0.2 = 0.3 exactly.

Part of the simple_* ecosystem of focused, single-purpose Eiffel libraries.

Quick Start

Installation

Add to your ECF file:

<library name="simple_decimal" location="$SIMPLE_DECIMAL/simple_decimal.ecf"/>

Basic Arithmetic

local
    a, b, total: SIMPLE_DECIMAL
do
    create a.make ("0.1")
    create b.make ("0.2")
    total := a + b
    print (total.out)  -- "0.3" (exact!)
end

Financial Calculations

local
    subtotal, tax_rate, tax, total: SIMPLE_DECIMAL
do
    create subtotal.make ("47.99")
    create tax_rate.make ("8.25")

    -- Calculate tax
    tax := (subtotal * tax_rate.from_percentage).round_cents
    total := subtotal + tax

    print (total.to_currency_string)  -- "$51.95"
end

Features

  • Arbitrary Precision - No floating-point errors
  • Financial Operations - Currency formatting, percentages, bill splitting
  • Multiple Rounding Modes - round_up, round_down, round_ceiling, round_floor, round_half_even
  • Immutable Operations - All arithmetic returns new values
  • Full Comparison - COMPARABLE interface for sorting
  • Currency Formatting - to_currency_string with commas and symbols
  • Percentage Calculations - add_percent, subtract_percent, as_percentage
  • IEEE 754-2008 compliant (via Gobo MA_DECIMAL)
  • Full Design by Contract support
